Los patrones de eventos se utilizan para modelar las interacciones entre los contratos inteligentes y su entorno, mejorando así la eficiencia, la seguridad y la flexibilidad de las aplicaciones basadas en tecnologías blockchain. El modelado de contratos inteligentes basado en patrones de eventos es una tendencia emergente. Definir un nuevo patrón, o seleccionar uno preexistente de un catálogo determinado, es fundamental para cualquier modelo de análisis que utilice patrones de eventos. Este proyecto presenta una propuesta para gestionar un catálogo de patrones de eventos aplicable al modelado conceptual de contratos inteligentes e incorpora técnicas de aprendizaje automático para optimizar la selección de patrones. PatCat, Pattern Catalogue, es la denominación de este prototipo que en la actual versión 3.0, se compone de una página html y un cuaderno NotbookLM (R) disponiblizados para uso público. La incorporación de patrones al inicio del proceso de modelado simplifica y clarifica la obtención de requisitos, entre otras ventajas, mientras que el uso del aprendizaje automático acelera la descripción del problema y la búsqueda de patrones en un repositorio científico validado. En consecuencia, una aplicación especializada para la gestión de un catálogo de patrones de eventos, con el apoyo de técnicas de aprendizaje automático, resulta útil para estandarizar y optimizar las tareas de modelado de contratos inteligentes.
